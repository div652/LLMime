# Usage Guide: LLM-to-Notion Universal Clipboard Bridge

Once the background `llm-bridge` daemon is running on your Linux machine, using it is completely seamless and invisible. 

## How It Works

Because the Daemon tracks your OS clipboard in the background, there is **zero friction**: no new keyboard shortcuts to memorize, and no clunky browser extensions.

### 1. Highlight LLM Output
Select the raw Markdown output generated by an LLM (such as ChatGPT, Claude, Gemini, or a local open-weights model). Make sure your selection contains structural technical tokens like:
- Block Math: `$$ E = mc^2 $$`
- Inline Math: `$O(N)$`
- Mermaid Flowcharts: ` ```mermaid`
- Syntax Highlighting: ` ```cpp`

### 2. Copy the Text
Press **`Ctrl + C`** just as you normally would. 

*Behind the scenes (in less than 50 milliseconds):* The daemon reads the copied Markdown buffer. Recognizing the technical payload, it dynamically compiles an equivalent HTML data structure on the fly and binds **both** the plaintext Markdown and the generated Rich HTML payload back to your clipboard.

### 3. Paste into your App
Navigate to Notion, Slite, or Obsidian and press **`Ctrl + V`**. 

- **In Notion / Slite:** The application will prioritize detecting the Rich HTML MIME data (`text/html`) inside the system clipboard. Upon parsing `notion-text-equation-block` or Mermaid `<pre>` structures, it instantiates native visual blocks rather than literally pasting the raw markdown string.
- **In Obsidian:** Since Obsidian primarily ingests pure markdown text, it gracefully falls back to catching the standard `text/plain` buffer. Your `$$` blocks are natively supported and will immediately render inside your notes.

## Troubleshooting
If you find the equations are pasting as plain raw strings:
1. Check that the script is running: `systemctl --user status llm-bridge.service`
2. Ensure you copied the raw Markdown properly, including the leading and trailing `$$` or ` ``` ` markers.
3. Check daemon execution logs: `journalctl --user -u llm-bridge.service -f`
